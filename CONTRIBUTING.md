# Contributing Guide

Спасибо за интерес к проекту! Ниже описаны основные правила и рекомендации.

## Подготовка окружения
1. Сделайте форк репозитория.
2. Клонируйте форк и настройте upstream-репозиторий.
3. Установите зависимости для backend и frontend согласно README.

## Работа с ветками
- Основная ветка: `main`.
- Для каждой задачи создавайте отдельную ветку вида `feature/описание-задачи` или `fix/кратко-о-багах`.
- Регулярно синхронизируйтесь с `main`, чтобы избегать конфликтов.

## Коммиты
- Используйте формат [Conventional Commits](https://www.conventionalcommits.org/):
  - `feat: добавить новую возможность`
  - `fix: исправить ошибку`
  - `chore: обновить зависимости`
- Делайте атомарные коммиты и понятные сообщения.

## Pull Request
1. Перед отправкой убедитесь, что проходят все тесты и линтеры.
2. Заполните шаблон PR полностью, приложите скриншоты при визуальных изменениях.
3. Ссылки на задачи и описания ожидаемого поведения обязательны.
4. Дождитесь ревью и оперативно реагируйте на комментарии.

## Код-ревью
- Будьте вежливы и конструктивны.
- Обсуждайте альтернативы и фиксируйте решения в комментариях или документации.

Благодарим за вклад!
