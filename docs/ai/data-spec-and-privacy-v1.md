# Data Spec & Privacy v1

## Источники данных
| Источник | Содержание | Ограничения |
| --- | --- | --- |
| Заказы | заголовок, описание, бюджет, категория, теги качества | PII режем, приватные вложения не индексируем |
| Профили | описание, навыки, ставки, рейтинги, завершённые работы | храним только опубликованные данные |
| Отклики | текст, ставка, ETA, метки успеха | PII-скраббер до сохранения в датасет |
| Чаты | только служебные метаданные (id, отметки времени, тип события) | контент не попадает в датасет без отдельного согласия |
| Исходы контрактов/споров | результат, причина, штрафы | агрегированные данные, без прямых цитат |

## PII-скраббер
- Удаляет телефоны (шаблоны +998, +7, 8-xxx), e-mail, Telegram/WhatsApp/почтовые хэндлы, внешние URL.
- Реализация: `backend/obsidian_backend/ai/pii_scrubber.py` с пайплайном `scrub_text`. Используем регулярные выражения и словарь замен.
- Пайплайн применяем перед записью текстов в витрину и датасеты для обучения.

## Retention и правовой базис
- Храним сырые тексты до 30 дней для расследований, затем только очищенные представления.
- Для обучения подсказок используем тексты по правовому основанию «исполнение договора» + обновлённые Terms (явное упоминание AI).
- Логи инференса анонимизируются и редактируются до PII-free формы, retention 90 дней.

## Принудительный opt-out
- В настройках профиля: чекбокс «Не использовать мои тексты для обучения подсказок». Значение хранится в профиле пользователя.
- Сбор датасетов читает флаг `profile.ai_training_optout`. Если ON, записи исключаются (фильтруем на уровне SQL view).
- Opt-out распространяется на все future датасеты; предыдущие данные удаляются по запросу в течение 30 дней.
