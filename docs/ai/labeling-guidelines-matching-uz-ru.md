# Руководство по разметке релевантности job↔profile (uz/ru)

## 1. Цель (Maqsad)
- **RU:** Оценить, насколько профиль специалиста подходит вакансии для задач матчинга и поиска.
- **UZ:** Vakansiya va mutaxassis profili o‘rtasidagi moslik darajasini matching/qidiruv modellari uchun baholash.

## 2. Единица разметки (Annotatsiya birligi)
Каждая запись = {вакансия, профиль}. Разметчик читает описание вакансии, требования, локацию и доступную историю профиля.

## 3. Шкала релевантности (Moslik shkala)
Используем 4 класса:
1. **Perfect / Mukammal (3)** — все ключевые навыки совпадают, опыт ≥ требований, язык/локация подходит.
2. **Good / Yaxshi (2)** — 70–90% навыков совпадает, опыт чуть ниже либо один минорный конфликт.
3. **Partial / Qisman (1)** — есть отдельные совпадения, но отсутствуют критичные навыки или сильный конфликт (локация, грейд).
4. **Irrelevant / Nomuvofiq (0)** — профиль вообще не подходит (другая сфера, нет базовых требований).

## 4. Обязательные поля в карточке разметки
| Поле | Описание |
| --- | --- |
| job_id/profile_id | уникальные идентификаторы |
| industry/function | проверяем совпадение направлений |
| skills checklist | чек-лист hard/soft навыков |
| language/location | требования и фактические данные |
| seniority | junior/middle/senior |
| заметки разметчика | краткое обоснование решения |

## 5. Процесс (Jarayon)
1. Прочитать вакансию и профиль целиком.
2. Сравнить hard skills, опыт по годам, язык, географию.
3. Выставить класс релевантности.
4. Добавить заметку (минимум 10 слов) с ключевыми причинами.
5. Второй разметчик перепроверяет 20% записей для расчёта согласия.

## 6. Примеры (Misollar)
- **Perfect:** Backend Go + Kubernetes 5 лет, вакансия требует 4 года и те же технологии.
- **Good:** Data Analyst Python/SQL 3 года, вакансия ищет 4+ года, но навыки совпадают.
- **Partial:** QA Manual пытается на вакансию QA Automation (нет кода).
- **Irrelevant:** Маркетолог на вакансию DevOps.

## 7. Edge-cases (Noan'anaviy holatlar)
- **Дублирующиеся вакансии:** отмечаем ID и не размечаем повторно.
- **Частичная удалёнка:** если вакансия onsite, а профиль remote-only, ставим максимум Partial.
- **Языковые требования:** отсутствие обязательного языка автоматически понижает класс до Partial.
- **Гэп в опыте >2 лет:** проверяем причину (курсы, декрет) и фиксируем в заметке.

## 8. Требования к объёму
- Минимум 1 000 пар (job, profile) для версии v1.
- Распределение классов: не менее 15% в каждом классе.

## 9. Контроль качества (Sifat nazorati)
- Двойная разметка 20% выборки.
- Cohen's κ ≥ 0.65.
- Еженедельный аудит 50 примеров лингвистом uz/ru.
