# Руководство по разметке споров и scam/spam (uz/ru)

## 1. Цель
Разделить входящие жалобы на классы для фильтров и арбитража.

## 2. Наборы классов
### A. Споры (Dispute classes)
1. **Payment / To'lov** — спор о выплатах, холдах, escrow.
2. **Scope / Vazifa** — несогласие по объёму работы, дедлайнам.
3. **Quality / Sifat** — претензии к качеству результата.
4. **Communication / Aloqa** — нарушения SLA коммуникаций, хамство.
5. **Policy / Siyosat** — жалобы на нарушения правил платформы.

### B. Scam/Spam classes
1. **Phishing / Fishing** — ссылки на вредоносные сайты, просьба данных.
2. **Fake job / Soxta vakansiya** — обещание работы без фактов, заведомо ложные вакансии.
3. **Mass spam / Ommaviy spam** — одинаковые сообщения >5 пользователям.
4. **Payment fraud / Moliyaviy firibgarlik** — требование предоплаты, переводы вне платформы.
5. **Benign / Xavfsiz** — ложное срабатывание.

## 3. Единица разметки
- Сообщение, цепочка чата или жалоба.
- Разметчик читает текст, вложения, метаданные (время, ID собеседников).

## 4. Процесс
1. Определить первичный класс спора или спама.
2. Отметить язык сообщения (ru/uz/eng/other).
3. Добавить теги: `payment`, `visa`, `crypto`, `in_person`.
4. Указать уровень риска (High/Medium/Low).

## 5. Примеры edge-case
- **Реклама курсов:** если пользователь продаёт курс, относится к Mass spam.
- **Просьба Telegram-контакта:** если цель — увезти в мессенджер для оплаты, классифицируем как Payment fraud.
- **Жалоба на отзыв:** если спор об отзыве, относим к Quality.
- **Мультиязычное сообщение:** классифицируем по содержанию, но отмечаем оба языка.

## 6. Требования к датасетам v1
- **Споры:** 500 примеров (минимум), минимум по 80 примеров на класс.
- **Scam/Spam:** 700 примеров, минимум 100 phishing, 150 fake job, 150 mass spam, 150 payment fraud, остальное benign.
- Баланс языков: 30% узбекских текстов, 30% смешанных.

## 7. Контроль качества
- Двойная разметка 25% примеров (разные эксперты).
- Cohen's κ ≥ 0.60 по каждой задаче.
- Автопроверка словаря (бан-лист слов). Несоответствия логируются.
