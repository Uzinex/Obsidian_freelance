# AI Stage Acceptance (Go/No-Go Checklist)

## 0. Общие данные
- [ ] Название модели / фичи: ___________________
- [ ] Версия модели / дата билда: _______________
- [ ] Ответственный владелец + on-call подтверждены.

## 1. Responsible AI & UX
- [ ] Пройдены фильтры: нежелательный контент, запрет контактов, юр/фин дисклеймеры.
- [ ] Кнопки «Пожаловаться» и «Отключить AI» доступны во всех UI-сценариях.
- [ ] Локализация uz/ru проверена (копирайт из UX copy library).
- [ ] Explainability-факторы логируются и отображаются в UI.

## 2. Тестирование
- [ ] Offline metrics ≥ целевых (precision@3, NDCG, coverage).
- [ ] Smoke-тесты промптов (safe checklist) прошли: RU/UZ, короткие/длинные/пустые запросы.
- [ ] QA подтвердила, что подсказки не ломают сетку и работают с i18n переключателем.
- [ ] Shadow-режим активирован (неделя логирования качества/latency без показа пользователю).

## 3. Наблюдаемость и безопасность
- [ ] Model health dashboard обновлён (latency p95, error rate, жалобы).
- [ ] Инцидентный канал и алерты подключены (latency, качество, фильтры контента).
- [ ] Логи объяснимости и жалоб сохраняются ≥ 30 дней.

## 4. Rollout план
- [ ] Описан трёхфазный запуск (shadow → 10–20% dark launch → 100%).
- [ ] Определены KPI и guardrails для переходов между фазами.
- [ ] Подготовлены acceptance protocol и rollout report шаблоны.
- [ ] Go/No-Go митинг назначен, участники подтверждены (AI, UX, Legal, Support).

## 5. Решение
- [ ] Все пункты 1–4 отмечены «выполнено».
- [ ] Риск-оценка ≤ допустимого уровня, при необходимости указаны компенсационные меры.
- [ ] Go/No-Go решение:
  - [ ] **Go** — переход к тёмному запуску.
  - [ ] **No-Go** — список блокеров и план устранения.
