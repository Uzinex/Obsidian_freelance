# Гайд по релизу

## 1. Подготовка ветки
1. Убедитесь, что основная ветка (`main` или `develop`) содержит все необходимые изменения.
2. Создайте ветку релиза с версией: `git checkout -b release/x.y.z`.
3. Обновите версионность в проекте (если требуется) и задокументируйте изменения в changelog.

## 2. Прогон тестов и проверок
```bash
# Backend
cd backend
pytest
python manage.py check

# Frontend
cd ../frontend
npm test
npm run lint
```
Убедитесь, что CI-пайплайны также прошли успешно.

## 3. Подготовка релизной сборки
1. Соберите frontend: `npm run build`.
2. Соберите Docker-образы или подготовьте артефакты деплоя согласно инфраструктурной инструкции.
3. Обновите версии зависимостей и миграции, если необходимо.

## 4. Выставление тега
1. Вернитесь в корень репозитория.
2. Создайте аннотированный тег: `git tag -a vx.y.z -m "Release x.y.z"`.
3. Отправьте ветку и тег в удалённый репозиторий: `git push origin release/x.y.z --tags`.

## 5. Деплой
1. Для staging окружения запустите процесс деплоя согласно внутренним инструкциям (CI/CD или ручной скрипт).
2. После проверки на staging выполните деплой на production.
3. Мониторьте метрики и логи, убедитесь в стабильности релиза.

## 6. Завершение
1. Создайте pull request из `release/x.y.z` в основную ветку.
2. После мерджа удалите ветку релиза локально и на remote.
3. Обновите документацию о релизе и уведомите команду.
