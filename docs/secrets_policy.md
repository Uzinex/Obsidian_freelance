# Политика управления секретами

## Хранение
- Рабочие значения переменных окружения не коммитятся в репозиторий.
- Продакшен-секреты для бэкенда хранятся в Railway в разделе `Variables` соответствующего сервиса.
- Секреты, необходимые для CI/CD, задаются в GitHub Actions Secrets и доступны только workflow-и с нужными правами.
- Локальная разработка использует файлы `.env`, основанные на шаблонах `backend/.env.example` и `frontend/.env.example`.

## Доступ
- Доступ к Railway и GitHub Secrets предоставляется только участникам команды, которым он необходим по роли.
- Любой доступ оформляется по принципу наименьших привилегий и пересматривается не реже одного раза в квартал.

## Ротация
- Ключи и пароли пересоздаются при любом подозрении на компрометацию и не реже одного раза в шесть месяцев.
- После ротации старые значения немедленно аннулируются, а команды уведомляются о необходимости обновить локальные `.env`.
- Все изменения секретов фиксируются в журнале (например, в тикет-системе или общем документе) с указанием причины и ответственного.
